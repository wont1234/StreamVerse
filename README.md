StreamVerse是一个的视频在线播放与互动网站项目，整体采用前后端分离架构实现。后端基于 Spring Boot 3 搭建，向前端提供统一的 REST API，并通过 WebSocket 实现实时交互能力；前端基于 Vue 3 + Vite + Vuetify 构建页面与交互逻辑。项目在功能上覆盖了从用户访问、内容上传、内容浏览到互动与后台运维的完整闭环，适合作为综合性 Web 应用开发与部署实践的示例。

在业务功能层面，系统支持用户登录鉴权、视频/图片等资源上传与管理、视频播放与互动等核心场景。后端通过 JWT 机制对用户身份进行签发与解析，在上传、发布弹幕、后台管理等关键操作中获取用户 ID 与角色信息，从而实现接口权限控制与行为可追溯。文件模块提供了视频、图片、头像等多类上传接口，上传完成后返回资源元数据，便于前端进行展示和业务绑定；同时系统提供对象存储相关配置管理能力，管理员可在后台维护 OSS 配置并触发客户端刷新，以适配不同的存储服务或部署环境。

项目的核心亮点之一是弹幕与实时推送链路。系统既提供弹幕的获取与发送接口，也通过 WebSocket 按视频维度管理连接会话并进行广播，从而实现“同一视频下多人实时可见”的弹幕体验。为了平衡互动体验与内容治理，弹幕保存环节引入了自动文本检测与可选审核策略：在弹幕入库前对内容进行过滤判定，必要时进入待审核流程，并结合站点开关决定审核强度，从工程上实现可配置的内容安全管控。

在工程化与可部署性方面，项目配套了 Docker Compose 部署方案，将 MySQL、Redis、后端服务与 Nginx（前端静态托管与 API/WS 反代）进行组合编排，支持一键启动与后续按模块重建更新。同时仓库中提供了较完整的部署说明文档，包含端口规划、环境变量配置、常见排障与公网发布思路，便于在本地或服务器环境中快速复现与演示系统能力。
